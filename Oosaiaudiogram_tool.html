Oosai Audiogram HTML Generator — Tool Guide
A complete audiometry workstation: enter values, draw on charts, auto-classify hearing loss, generate HTML output, and download professional PDF prescriptions.
1 · Entering Audiogram Values
Type mode (default): Click any cell in the Audiogram Values grid and type the threshold in dB. Press Tab to move across frequencies.
NR checkbox: Tick the tiny NR box below a cell if the patient had no response — the symbol will be plotted with a downward arrow at the top of the chart.
Yellow cells = PTA frequencies used for the average (500 Hz, 1k, 2k by default).
ABG rows at the bottom of the grid are read-only and auto-calculated as AC − BC at each frequency.
2 · Draw on Chart Mode
In the Draw Mode panel on the left sidebar, click 🎯 Draw on Chart.
Select the Row to draw on (e.g. RE — AC Unmasked) from the dropdown.
Click or drag on the chart canvas to place values — the tool snaps to the nearest frequency column and rounds dB to the nearest 5.
A ghost crosshair shows a live preview tooltip with frequency and dB before you click.
Switch back to ✏️ Type Values mode to return to normal typing. Both modes update each other in real time.
3 · PTA Formula & Source Selection
Standard (500+1k+2k) ÷ 3 — WHO/clinical default.
Fletcher (500+1k+2k+4k) ÷ 4 — includes 4 kHz.
Use the AC PTA from dropdowns to choose whether to calculate PTA from unmasked or masked AC rows for each ear.
Use the BC PTA from dropdowns similarly for bone conduction.
ABG average always uses 500+1k+2k regardless of the PTA formula — this is the clinical standard for air-bone gap classification.
4 · Live Classification (Sidebar)
The sidebar updates in real time as you type. The panels show:
PTA Average & ABG pills — LT and RT PTA in dB with degree (Mild/Moderate/etc.), plus the averaged ABG with significance flag.
Hearing Loss Classification — auto-classifies each ear using:
Normal: BC<20, AC<20, ABG ≤10 dB
Conductive: BC<20, AC≥21, ABG >5 dB
Mixed: BC≥21, AC≥21, ABG >15 dB
SNHL: BC≥21, AC≥21, ABG ≤15 dB
📋 Prescription Reference box (yellow) shows the full combined label with dB range — for your internal reference only, not included in the HTML output.
5 · SRT, TDT, STAT, Rinne & Weber
Enter SRT (Speech Reception Threshold in dB), TDT result (Normal/Abnormal), and STAT result (Pass/Fail) for each ear.
Rinne Test: Enter result per ear — Positive (AC>BC), Negative (BC>AC), or False Negative.
Weber Test: Enter under the RE row only (Weber is a midline tuning fork test, not per-ear).
Use the Include in output toggle to control whether Rinne/Weber appears in the generated HTML.
6 · Generating HTML Output
Click ⚡ Generate HTML Output at the bottom of the sidebar.
The output appears in the right panel — click Copy HTML and paste it into your records system description field.
Output Options (in the Draw Mode box) control what's included:
• Show ABG gap shading on chart
• Include ABG rows in the output table
• Include type classification & report lines
The dB range (e.g. "21–40 dB HL") is intentionally excluded from the HTML output — it only shows in your sidebar Prescription Reference box.
7 · Prescription Records & PDF
Click 📋 Prescriptions in the top bar to open the Prescription system.
New Prescription tab: Fill in patient details, audiometer model, and confirm or override the auto-filled clinical summary. Clinic info and Terms & Conditions are pre-filled but editable. Click 💾 Save Prescription.
Records tab: Search by order # or patient name. Click a record to select it, then use 📥 Load to Audiogram to restore all values to the grid, or 🖨️ Download PDF to generate the prescription PDF.
The PDF opens in a new browser tab/window — use the browser's built-in Save as PDF or Print to save it.
Records are stored in your browser's localStorage. Use Backup / Restore to export JSON for safekeeping.
8 · Backup & Restore
Download JSON Backup — exports all saved prescriptions as a dated JSON file. Do this regularly to protect your data.
Import JSON — merges a backup file into the current records. Records with the same Order # are updated; new records are added. Existing records with different Order #s are kept.
